## Fios soltos

- `.codex` continua não rastreado. É arquivo local/sistema e deve permanecer fora do commit.
- `logoVeredaok.psd` está não rastreado. É arquivo-fonte do logo, com cerca de 80 KB. Decidir amanhã se entra em `assets/source/`, se fica fora do repo ou se vai para armazenamento externo.
- Ainda não houve QA visual com navegador real depois das paletas. A sintaxe passou, mas precisamos olhar contrastes e estados em tela.
- As paletas escuras podem revelar pontos com `#fff` hardcoded ou transparências claras demais. A troca por variáveis já cobriu bastante, mas precisa inspeção visual.
- O Espelho de Voz está funcional, mas os léxicos ainda são pequenos. Bom para primeiro corte; próximo passo é ampliar com cuidado.
- A busca do Guia de Ofícios é útil, mas ainda não busca dentro de campos profundos dos guias, como `guidance.sections`, `reminders` e `model.references`.
- Os atalhos do manifest para “Manuscrito” e “Academia” ainda abrem a mesma URL base. Para atalhos reais, seria ideal aceitar hash ou query na inicialização.
- Checklists persistentes por projeto ainda não foram implementados.
- O botão “A partir do guia” ainda pode ser pouco claro para usuário novo, porque depende do guia selecionado.
- O nome “Manuscrito” na barra superior ainda pode ser revisado para “Editor”, conforme roadmap.

## Riscos atuais

- `app.js`, `styles.css` e `template-engine.js` estão grandes. Ainda aceitável para MVP, mas novas features devem preferir engines separadas quando possível.
- O produto está ganhando muitas capacidades rapidamente. Precisamos manter logs e roadmap atualizados para não perder coerência.
- O Espelho de Voz não deve ser vendido como estilometria científica. A UI já inclui disclaimer, mas isso precisa ser preservado em futuras iterações.
